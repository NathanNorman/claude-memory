## ADDED Requirements

### REQ-AP-1: Promotion Trigger
- A sequence with status `observed` and frequency >= configurable threshold (default: 5) MUST be eligible for promotion
- `ratchet promote` MUST accept a sequence ID or auto-select the highest-frequency eligible sequence
- Promotion MUST NOT proceed if the target brew-skill repo has uncommitted changes on its main branch

### REQ-AP-2: Code Generation
- The promoter MUST create a git worktree of the target brew-skill repo
- The promoter MUST invoke Claude Code (via `claude --print` or similar) with a structured prompt containing:
  - The detected sequence (method names, typical arguments, frequency)
  - The existing `client.py` source (for style and convention matching)
  - The existing SKILL.md and reference files
  - Instructions to: write the new method, add a test, update SKILL.md (remove from Common Patterns, add to reference), update quick-reference.md
- All changes (code + skill + tests) MUST be in a single commit on a new branch

### REQ-AP-3: PR Creation
- The promoter MUST push the branch and open a PR via `gh pr create`
- The PR title MUST follow the pattern: `ratchet: promote <method_name> from <sequence_description>`
- The PR body MUST include: the detected sequence, frequency data, session count, and a note that it was auto-generated by ratchet
- The PR MUST NOT be auto-merged — human review is required

### REQ-AP-4: Status Tracking
- After PR creation, the sequence status MUST be updated to `promoted` with the PR URL stored in `promoted_pr`
- `ratchet status` MUST show promoted sequences and their PR status
- If a promoted PR is closed without merging, the sequence status MUST revert to `observed` (eligible for re-promotion)
